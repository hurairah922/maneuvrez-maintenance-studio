## 2. Product Promise

Maintenance Mode Studio turns a closed, hidden, or unfinished WordPress site into a modern interactive landing page.

Instead of showing a static maintenance message, the plugin lets visitors:

- play a quick game
- answer MCQs
- submit feedback
- join a waitlist
- contact the site owner
- follow social links
- access login or password-protected preview flows

The admin gets a complete WordPress-native system to design, control, preview, and publish these experiences.

## 3. Core Product Principle

This is not a general page builder.

This is a focused experience builder for maintenance, coming soon, launch, private site, and temporary site states.

The plugin must stay useful, fast, safe, and simple for non-technical admins.

## 6. User Experience Standard

The frontend must feel:

- modern
- premium
- playful where appropriate
- responsive
- fast
- interactive
- polished
- trustworthy

The default experience must look good without configuration.

## 7. Admin Experience Standard

The admin panel must support two usage levels:

### Simple Mode

For non-technical admins:

- select mode
- select template
- edit copy
- enable/disable components
- update colors/logo
- preview
- publish

### Advanced Mode

For power users:

- drag components across allowed layout zones
- edit component settings
- create custom fields
- add custom CSS
- add custom JS if allowed
- save custom layouts
- manage bypass links
- manage submissions
- manage leaderboard

## 8. Builder Boundary

The plugin may include a drag-and-drop mini-builder.

The plugin must not become a full freeform page builder in V1.

Allowed builder model:

- fixed layout zones
- draggable compatible components
- template presets
- live preview
- saved custom layouts

Not allowed in V1:

- arbitrary absolute positioning
- nested page-builder sections
- full Gutenberg clone
- Elementor-style editing system
- complex responsive overrides per element

The free plugin must be complete and useful by itself.

The architecture should allow a future Pro add-on to extend:

- advanced templates
- extra games
- advanced leaderboards
- CRM integrations
- email marketing integrations
- conditional form logic
- advanced analytics
- page-level targeting
- WooCommerce-specific rules

- saved template cloud library

V1 must not include payment, license validation, subscription billing, or locked feature prompts that harm WordPress.org acceptance.

The plugin should rank for:

- maintenance mode
- coming soon
- WordPress coming soon page

## 12. Non-Negotiables

- The page must never look broken on small screens.
- The plugin must work without external services by default.
- All external integrations must be optional.
- The frontend must remain lightweight.
- Admin controls must stay clear.
- Every saved value must be sanitized.
- Every rendered value must be escaped.
- Every state-changing request must use permissions and nonces.
- The plugin must be WordPress.org-friendly from day one.
